Revenue & ROAS calculation
ROAS is calculated from actual purchase revenue reported by your Meta pixel (action_values → purchase). This is per-creative, per-campaign, per-country — accurate by default.

How ROAS is calculated: Revenue ÷ Spend — using the exact purchase value your Meta pixel fires on each order. No fixed multiplier. If your pixel doesn't pass value in the Purchase event, connect your store or pass it manually in the pixel code: fbq('track','Purchase',{value:29.99,currency:'EUR'})

Meta Ads API
Connect your Meta ad account to sync performance data automatically — no more CSV exports.
How to get your credentials:
1. Go to developers.facebook.com/apps → Create App → Business type
2. Note your App ID from the app dashboard
3. Go to Graph API Explorer → select your app → Add permissions: ads_read read_insights ads_management
4. Click Generate Access Token → copy the token
5. Exchange for a 60-day token at: graph.facebook.com/oauth/access_token?grant_type=fb_exchange_token&client_id=APP_ID&client_secret=APP_SECRET&fb_exchange_token=SHORT_TOKEN
6. Your Ad Account ID is in Meta Business Manager → Ad Accounts — format: act_123456789

How it works: Once connected, the dashboard pulls your ad data directly via Meta's Marketing API every time you open the Meta Ads tab or click Sync. Performance data (spend, impressions, CTR, CPC, purchases) maps automatically to creatives using the WW_ naming convention. You can still upload a CSV below as a fallback.
